# 03 · Decisions

| Date | Title | Context | Decision | Consequences |
|---|---|---|---|---|
| 2026-05-29 | One launcher, swappable backend | Daemon only needs an OpenAI-shape `/v1/audio/transcriptions` HTTP endpoint; Docker dep lived only in launcher scripts | Add single `run.sh` that detects host and brings up a per-platform backend behind the same `WHISPER_API` | No client rewrite; backends are independent and individually testable |
| 2026-05-29 | Drop Docker on Mac | Docker on macOS can't reach the Apple-Silicon GPU → CPU-only in a RAM-heavy VM, bad on 8 GB | Mac uses native whisper.cpp + Metal, not Docker | Mac needs a native install path (brew/cmake) instead of `docker run` |
| 2026-05-29 | whisper.cpp = shared Linux+Mac artifact | Need to honor red/green TDD but dev box is Linux; Mac-specific runtimes can't be tested here | Linux-no-GPU backend uses whisper.cpp CPU — the *same* server Mac runs with Metal | Developing CPU backend here de-risks Mac to one build flag; mlx can't share this |
| 2026-05-29 | Keep Linux+NVIDIA on Docker | Existing CUDA container path is proven and in daily use | Leave it unchanged; `run.sh` routes Linux+GPU → Docker | No regression risk to the working path |
| 2026-05-29 | mlx-whisper deferred to Phase 2 | Fastest on Apple Silicon but `import mlx` fails on Linux and it ships no HTTP server | Evaluate on-device only; benchmark vs whisper.cpp on the real M1 before making it default | Phase 1 default stays whisper.cpp (safe drop-in); default revisited after benchmark |
| 2026-05-30 | Mac default model = mlx large-v3-turbo q4 | M1 8 GB is tight, and the non-quantized mlx model download is too large/slow for first-run smoke tests | `mlx-community/whisper-large-v3-turbo-q4`; disable HF Xet by default for predictable downloads | Comfortable default for Apple Silicon; can override with `WHISPER_MLX_MODEL` |
| 2026-05-30 | Ship a generated Mac `.app` wrapper first | Users should grant permissions to a named app, not Terminal/iTerm/VS Code; but a full Swift menu bar app would slow the current install-validation loop | Generate `~/Applications/Whisper Hotkey.app` with an `Info.plist` and launcher that runs the existing repo `run.sh` | Permissions can appear under **Whisper Hotkey** and the CLI remains source of truth. Future Swift/menu-bar/signing work can replace the wrapper without changing backend logic |
| 2026-05-30 | Provide a first-party uninstall script | The installer creates user-visible app/state/log files and mlx-whisper downloads large HuggingFace models lazily; users need a supported cleanup path | Add `uninstall.sh` plus `tests/test_install_uninstall.sh` for fake-mac install/uninstall behavior | Keep cache deletion scoped to known project model dirs by default; never wipe all HuggingFace or Pixi state unless the user opts in |
| 2026-05-30 | Rename to `tigris-whisper` only after repository coordination | The bootstrap one-liner depends on the GitHub slug; changing `REPO_SLUG` before the repo rename would break clean installs | Track rename as Phase 4.6 and perform user-facing/app/docs changes together with the GitHub repo rename | Internal Python module filenames can remain until there is a technical reason to rename them |
| 2026-05-29 | Candidate future backends (Parakeet / Voxtral / Windows) | User asked what it'd take to add NVIDIA models, Voxtral, and Windows | Defer to Phase 3; all reuse the `/v1/audio/transcriptions` plug-in pattern. Parakeet-0.6B = best Linux+GPU upgrade; Voxtral only via **4-bit quant** on this 8 GB-VRAM box; Windows easiest under WSL2 | No commitment now; tracked in `01_plan.md` Phase 3. None address the Mac goal |
| 2026-05-29 | **Mac default switched to mlx-whisper** (was whisper.cpp Metal) | Wanted to avoid the ~1–2 GB Xcode CLT + Homebrew + binary-distribution burden. mlx-whisper installs as prebuilt wheels via uv — no compiler, no brew, no binary to ship; model auto-downloads from HF | Darwin → `mlx`; whisper.cpp Metal kept as `WHISPER_BACKEND=whispercpp_metal` fallback | **Loses Linux-side TDD of the Mac transcription** (mlx is Apple-Silicon-only); we test the HTTP wrapper with a mocked boundary instead. `git clone` still pulls CLT on a fresh Mac (tarball install = open follow-up) |
